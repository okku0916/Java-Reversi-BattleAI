# Java Reversi BattleAI

このプロジェクトは、Javaで作成されたリバーシ（オセロ）の対戦用AIのソースコードです。
GitおよびGitHubに慣れていない方向けに、このコードのダウンロード方法や更新方法を以下にまとめています。

## 📋 事前準備 (Gitのインストール)

このコードを手元のパソコンで扱うには、**Git (ギット)** というツールが必要です。
まだインストールされていない場合は、以下の公式サイトからインストールしてください。

- [Git for Windows / Mac のダウンロード](https://git-scm.com/downloads)
もしくは
```bash
brew install git
```

## 💻 VS Codeを使った操作方法 (推奨)

黒い画面（ターミナル）を使わず、VS Codeのボタン操作だけで完結する方法です。

### 📥 1. コードをダウンロードする (Clone)

1.  VS Codeを起動します。
2.  左側のメニューにある「ソース管理アイコン」（枝分かれした線のマーク）をクリックします。
3.  **「リポジトリのクローン (Clone Repository)」** というボタンをクリックします。
4.  画面上部に出てくる入力欄に、以下のURLを貼り付けてEnterキーを押します。
    ```text
    [https://github.com/PhenylPropion/Java-Reversi-BattleAI.git](https://github.com/PhenylPropion/Java-Reversi-BattleAI.git)
    ```
5.  保存するフォルダを聞かれるので、デスクトップやドキュメントなど好きな場所を選択します。
6.  ダウンロードが終わると右下に「開きますか？」と出るので「開く (Open)」を選択してください。



### 🔄 2. 最新の状態に更新する (Pull)

他の人がコードを更新した際、手元のファイルを最新版に合わせる手順です。

1.  VS Codeの左下（ステータスバー）にある、**回転する矢印アイコン**（または雲のアイコン）に注目してください。
2.  ここに数字が出ている場合、新しい変更があります。
3.  このアイコンをクリックすると、最新のコードがダウンロードされ、手元のファイルが更新されます。
    * または、左側の「ソース管理」メニューの「...」ボタン → 「プル (Pull)」を選択しても同じことができます。



### 🛠 3. コードを変更してアップロードする場合 (Push)

> **⚠️ 注意:** このリポジトリの管理者は私だけです。
> 直接アップロード（Push）しようとするとエラーになります。自分の変更を保存したい場合は、GitHub上で「Fork」してから行ってください。

もしFork済みの自分のリポジトリで作業している場合は、以下の手順で保存できます。

1.  ファイルを保存すると、左側の「ソース管理」アイコンに数字がつきます。
2.  ソース管理パネルの入力欄（メッセージ欄）に、「バグ修正」などのメモを入力します。
3.  **「コミット (Commit)」** ボタン（またはチェックマーク）を押します。
4.  **「変更の同期 (Sync Changes)」** ボタンを押すと、GitHub上にアップロードされます。

## 🤝 変更を管理者に送る方法 (Pull Request)

もしあなたがコードの改善点を見つけたり、新機能を追加したりして、それをこのプロジェクト（管理者）に取り込んでほしい場合は、**「プルリクエスト」**という機能を使います。

管理者に「私のコードを採用してください！」と依頼する一連の流れは以下の通りです。

### 手順 1: リポジトリを自分のアカウントにコピーする (Fork)

あなたは直接このリポジトリに書き込む権限がありません。まず、自分のアカウントへコピーを作ります。

1.  GitHubのこのページ右上の **「Fork」** ボタンをクリックします。
2.  「Create fork」をクリックすると、あなたのアカウント内に同じリポジトリがコピーされます。



### 手順 2: コピーしたコードをVS Codeで開く

1.  **自分のアカウントにできた** リポジトリのページから、URLをコピーします（緑色のCodeボタン）。
2.  VS Codeを開き、「リポジトリのクローン」からそのURLを入力してダウンロードします。

### 手順 3: 作業用の「ブランチ」を作る (推奨)

メインのコード（main）を汚さないように、作業専用の場所（ブランチ）を作ります。

1.  VS Codeの左下にある **「main」**（または master）と書かれた文字をクリックします。
2.  画面上部に出るメニューから **「新しいブランチの作成 (Create new branch)」** を選びます。
3.  わかりやすい名前を付けます（例: `fix-bug`, `add-new-ai`）てEnterを押します。
4.  左下の文字が、入力した名前に変わったことを確認してください。



### 手順 4: 変更して保存（コミット & プッシュ）

コードを編集し、VS Codeの「ソース管理」画面からコミット（記録）と同期（プッシュ）を行います。
※この操作で、**あなたのGitHubアカウント**にあるリポジトリに変更が保存されます。

### 手順 5: プルリクエストを送る

いよいよ管理者に変更を通知します。

1.  ブラウザで **GitHubの自分のリポジトリ** のページを開きます。
2.  画面上部に黄色い枠で **「Compare & pull request」** というボタンが表示されているはずなので、クリックします。
3.  タイトルと本文（何を変更したか）を記入し、緑色の **「Create pull request」** ボタンを押します。



これで管理者へ通知が届きます！あとは管理者が内容を確認し、問題なければ本流のコードに統合（マージ）されます。

## 🔄 よくある質問 (Q&A)

**Q. 「Fork」と「Clone」は何が違うの？**
* **Fork (フォーク):** GitHub上で、人のプロジェクトを自分のアカウントにコピーすること。（クラウド上のコピー）
* **Clone (クローン):** GitHub上のコードを、手元のパソコンにコピーすること。（PCへのダウンロード）
    * 他人のコードを修正したい時は **「Fork」してから「Clone」** します。

**Q. ブランチって何？**
* 「セーブデータの別スロット」のようなものです。
* `main` ブランチが「本番用」、`fix-bug` ブランチが「実験用」のように使い分けることで、失敗しても安全にやり直せます。
